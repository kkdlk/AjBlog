接口是一种规范：符合规范就可以使用此类


使用抽象方法去定义一组功能

必须实现者给提供方法的实现



接口的定义 

	interface 接口名称{

		}
接口是一个特殊的抽象类

	默认会有一个abstract（抽象类）


注意点：
     1.接口没有构造器，不能创建对象。（接口不能new）
     2.接口中定义变量，都是全局的静态常量。
     3.接口中的方法，都是公共的抽象的方法
     4.接口中的内部类，都是公共的抽象的类
     5.接口可以继承，他可以多继承
	interface 接口名称 exdents 继承接口名,继承接口	名{


	}（类是不能多继承的，接口继承的时候不会去覆盖父类接口的方法）




	接口的实现类
	类后加implements 接口
	实现类必须覆盖接口类的方法
	（覆盖的时候，子类方法权限要比父类大。或者一样大）
